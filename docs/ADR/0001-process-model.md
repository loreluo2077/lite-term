# ADR-0001：第一阶段采用“每会话一个子进程”

## 状态

Accepted

## 背景

项目第一阶段仅做本地终端，但要求从一开始保留未来扩展到 SSH 的架构空间，同时希望会话隔离清晰、便于自动化验证与故障定位。

## 决策

采用“每个终端会话一个 session worker 子进程”的进程模型。

会话子进程负责：
- 会话专属 WS 服务
- session adapter（local）
- 会话生命周期与数据流

Control plane 负责：
- 会话创建/销毁
- 端口分配
- 进程管理
- registry

## 理由

1. 故障隔离更强
2. 后续 SSH/其他协议扩展可沿用同一模式
3. 测试边界更清晰（可针对单会话 worker 做集成测试）
4. 避免 Electron Main 进程承担终端协议复杂度

## 代价

1. 端口与进程管理复杂度增加
2. spawn 成本高于单进程多会话
3. 需要更严格的回收测试

## 后续约束

- 第一阶段必须建立 orphan worker 清理机制
- 测试必须验证 worker 回收
