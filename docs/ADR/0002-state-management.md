# ADR-0002：前端状态管理使用 Jotai

## 状态

Accepted

## 背景

第一阶段前端功能很少（tabs + terminal panes + session status），但后续会扩展到 SSH、设置、终端增强。需要一种足够轻量、又便于按功能拆分的状态管理方式。

## 决策

使用 `jotai` 作为前端状态管理方案。

## 理由

1. 原子化模型适合按会话拆分状态（`atomFamily`）
2. 学习和维护成本低
3. 与 React 组合自然，适合渐进扩展
4. 相比大型全局 store，更容易在第一阶段保持简单

## 非目标

- 不把 xterm 实例直接放入 atom
- 不在第一阶段构建复杂全局事件总线

## 后续约束

- 会话元数据、UI 状态、终端偏好状态分层管理
- 前后端协议数据优先通过 `packages/shared` 类型约束
